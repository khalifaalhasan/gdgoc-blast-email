=========================================================
PRODUCT REQUIREMENTS DOCUMENT (PRD)
GDGoC Bulk Email & Certificate Distribution Web App
=========================================================

1. LATAR BELAKANG & TUJUAN
Sistem saat ini berbasis script Python lokal yang rentan terhadap masalah koneksi/sinyal dan membutuhkan pengeditan kode manual (hardcode) pada template.py dan config.py untuk setiap event.
Tujuan dari project ini adalah membuat aplikasi berbasis Web (GUI) yang di-hosting di VPS agar proses blast email berjalan di server yang stabil, serta memiliki antarmuka yang memudahkan panitia/user untuk membuat campaign, mengunggah data, dan mengatur template tanpa menyentuh kode.

2. FITUR UTAMA (CORE FEATURES)
a. Manajemen Template Dinamis (Dynamic Template Builder)
   - UI untuk menginput Subjek Email yang bisa berubah tiap event.
   - WYSIWYG Editor atau Text Editor dengan dukungan variabel dinamis (contoh: {{nama}}, {{role}}, {{event_name}}) untuk Body Email.
   - Preview email sebelum dikirim.

b. Integrasi Data Penerima (CSV Upload)
   - Form upload file CSV yang berisi daftar penerima (Nama, Email, Role, dll).
   - Validasi otomatis format CSV (memastikan kolom Nama dan Email ada).

c. Integrasi Sertifikat dari Google Drive
   - Input field untuk memasukkan Link Folder Google Drive yang berisi sertifikat peserta.
   - Sistem akan secara otomatis membaca isi folder tersebut via API, dan mencocokkan nama file sertifikat (misal: *.pdf / *.png) dengan nama peserta di dalam file CSV.

d. Background Processing & Blast Engine
   - Sistem pengiriman email berjalan di background (background worker/job queue) di VPS.
   - Anti-freeze UI: User bisa menutup browser dan proses blast tetap berjalan di server.
   - Terdapat jeda (delay) anti-spam otomatis.

e. Dashboard & Monitoring (Real-time Tracker)
   - Halaman dashboard untuk melihat progress blast email (Berapa % selesai, jumlah Sukses, jumlah Gagal).
   - Log error jika sertifikat tidak ditemukan atau email tidak valid.

3. ALUR PENGGUNA (USER JOURNEY)
   1. User membuka Web App dan membuat "Campaign Baru" (Contoh: "Blast Sertif InspireHer 2026").
   2. User mengedit Subjek Email dan Body Email di Template Editor, menggunakan variabel seperti {{nama}}.
   3. User mengunggah file CSV daftar peserta.
   4. User memasukkan Link Folder Google Drive yang berisi file sertifikat.
   5. Sistem memvalidasi akses Drive dan memunculkan konfirmasi jumlah data yang match antara CSV dan Drive.
   6. User klik "Start Blast".
   7. UI menampilkan progress bar secara real-time. Proses berjalan aman di VPS.

4. REKOMENDASI ARSITEKTUR TEKNOLOGI
Mengingat basis script yang sudah ada adalah Python, berikut rancangan stack yang disarankan:
- Backend & API: Python (FastAPI / Flask) - Memudahkan migrasi kode GMail API & pencarian file yang sudah ada.
- Background Worker: Celery + Redis (Untuk menjalankan antrean pengiriman email di background).
- Frontend: HTML/Tailwind CSS dengan Vanilla JS, ATAU React/Next.js jika ingin UI yang lebih modern.
- Integrasi: Google Drive API (Service Account) untuk membaca dan mendownload sertifikat secara real-time, GMail API untuk mengirim email.
- Deployment: Docker & Docker Compose untuk kemudahan deploy di VPS (menjalankan Web, Worker, dan Redis sekaligus).
